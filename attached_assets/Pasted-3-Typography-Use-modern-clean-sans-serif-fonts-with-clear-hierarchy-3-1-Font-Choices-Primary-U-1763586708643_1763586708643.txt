3. Typography

Use modern, clean sans-serif fonts with clear hierarchy.

3.1 Font Choices

Primary UI font:

Prefer: Inter, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif

Monospace (if needed):

ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace

3.2 Type Scale (Desktop defaults)

H1: 30–32px, font-semibold, text-ttf-navy

H2: 24–26px, font-semibold, text-ttf-navy

H3: 20px, font-semibold, text-ttf-grey-900

Section title / card title: 16–18px, font-semibold

Body: 14–16px, font-normal, text-ttf-grey-700

Caption / helper: 12–13px, text-ttf-grey-500

3.3 Usage Rules

Limit to two weights in most screens: font-normal and font-semibold.

No all-caps for long labels. Only short tags/badges (e.g. APPROVED, DRAFT).

Keep line-height comfortable (leading-relaxed or leading-normal).

4. Layout & Spacing
4.1 Spacing Scale (8px base)

Use an 8px rhythm:

4, 8, 12, 16, 20, 24, 32, 40 mapped to Tailwind: 1, 2, 3, 4, 5, 6, 8, 10 etc.

Rules:

Cards: p-4 or p-6

Page padding: p-6 on mobile, p-8 on desktop

Gaps between form fields: space-y-4

Gaps between cards/sections: space-y-6 or gap-6

4.2 Layout

Main layout: Left sidebar + top bar pattern.

Content max width: use max-w-6xl or max-w-7xl mx-auto for main content.

Avoid edge-to-edge cramped layouts; always maintain breathing room.

4.3 Radii & Shadows

Border radius: rounded-xl or rounded-2xl for cards and modals.

Buttons: rounded-full or rounded-lg (but consistent across the app).

Shadows: use soft single-layer shadows like shadow-sm or shadow-md. No heavy drops.

Example card style:

<div className="bg-white rounded-2xl shadow-sm border border-ttf-gray-100 p-6">
  {/* card content */}
</div>

For all components, favour clarity over “flashiness”.

Borders: border border-ttf-gray-300

Focus: ring-2 ring-ttf-aqua border-ttf-aqua

Radius: rounded-lg

Label: 14px, font-medium, text-ttf-grey-700

Helper/error text: 12–13px, muted colours.

5.3 Navigation

Top Bar:

Background: bg-ttf-navy

Text/icons: white or ttf-lightSky

Show app name and key shortcuts (Search, Profile).

Sidebar:

Background: white or very light (ttf-lightSky)

Active item: pill with bg-ttf-blue/10 and left border or small bar in ttf-blue.

Example sidebar item:

<button

5.4 Cards & Dashboards

Use cards with clear titles and small descriptions.

Use consistent layout for dashboards:

Top row: metrics (Total requests, Pending approvals, Estimated savings, etc.)

Middle: charts (approvals over time, cost savings)

Bottom: tables / recent activity.

Metric card example:

<div className="bg-white rounded-2xl shadow-sm border border-ttf-gray-100 p-4 flex flex-col gap-1">
  <span className="text-xs font-medium text-ttf-grey-500 uppercase tracking-wide">
    Estimated Savings this Month
  </span>
  <span className="text-2xl font-semibold text-ttf-navy">
    $12,450
  </span>
  <span className="text-xs text-ttf-success font-medium">
    +18% vs last month
  </span>
</div>

5.5 Tables

Use striped or lightly separated rows: border-b border-ttf-gray-100.

Header row: bg-ttf-lightSky, text-xs font-semibold text-ttf-grey-700.

Row hover: hover:bg-ttf-lightSky.

5.6 Badges & Status Pills

Use small rounded-full pills:

<span className="inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-medium bg-ttf-lightSky text-ttf-blue">
  Pending Approval
</span>


Status colours:

Approved: bg-ttf-success/10 text-ttf-success

Rejected: bg-ttf-error/10 text-ttf-error

Pending: bg-ttf-lightSky text-ttf-blue

5.7 Alerts & Banners

Info: bg-ttf-lightSky border border-ttf-aqua/30 text-ttf-navy

Warning: bg-ttf-warning/10 border border-ttf-warning/50 text-ttf-warning

Error: bg-ttf-error/10 border border-ttf-error/40 text-ttf-error

6. Charts & Data Visualisation

Use simple, clean charts (bar, line, donut).

Primary chart colour: #0057B8 (blue).

Secondary series: #1FBED6, #002A52, and neutral greys.

Avoid bright red as main chart colour; reserve red for negative metrics.

Ensure charts:

Have clear legends & axis labels.

Use text-ttf-grey-700 for labels.

Use minimal grid lines (soft grey).

7. Microcopy & Tone

Short, clear, action-driven labels:

“Submit for approval”, “Return to requester”, “Download report”

Helper text is concrete:

“This trip exceeds policy by 12% due to fare class and hotel rate.”

Avoid jargon-heavy phrases; explain travel/policy jargon in tooltips where necessary.

8. Implementation Rules For You (Replit AI)

When creating or editing components, always use the brand colours and typography described above.

Replace arbitrary Tailwind colours (blue-500, gray-200, etc.) with the ttf.* palette.

Normalise spacing to the 8px scale and consistent card/button patterns.

Make dashboards and tables reflect this design without changing core business logic.

Keep everything looking like a single, cohesive enterprise app—not a mix of random templates.

Apply these rules consistently across:

Layout (main shell)

All forms (travel request, approvals, admin settings)

Dashboards & reports

Modals and wizards

Any new pages you create.
